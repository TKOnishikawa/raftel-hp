A: X Cross Slash — 仕様

コンセプト2本の対角線が交差してX字を描く。LayerXロゴの「X」を直接引用。セクション進入時にシャキーンと描画。最もストレートな幾何学表現。
SVG構造viewBox内の対角pathが2本1組。stroke-dasharray/dashoffset で描画アニメーション。
ストロークメイン: amber→rose グラデ / 1.5px。サブ: blue / 0.5px / opacity 0.4。
タイミング0.7s / expo-out。2本目は0.15s遅延でstagger。背景X字は0.3-0.4s遅延。
トリガーIntersectionObserver (threshold: 0.15)。一度だけ発火。
面白さ / テクノロジー感★★★★ / ★★★★ — ダイナミックかつ意味がある（omoshikuのクロス=統合の象徴）。

B: Angular Brackets — 仕様

C: Diagonal Rain — 仕様

D: Network Grid — 仕様

コンセプト直交するグリッド線が描画され、交差点にノード（点）とミニXが現れる。データネットワーク / 基盤の表現。「仕組み」の可視化。
SVG構造縦線3本 + 横線2本（グリッド）→ 交差点に circle ノード → ミニX path。3層の時間差。
ストロークグリッド: blue 20% / 0.5px。アクセントX: rose / 1.5px。ノード: blue fill / r:3-5px。
タイミンググリッド線: 0.5-0.8s。ノード: 0.4s (r=0→target, 0.5s delay)。ミニX: 0.3s (0.6s delay)。
面白さ / テクノロジー感★★★★ / ★★★★★ — 最もデータ/インフラ的。交差点のXがomoshikuのクロスを反復。

E: Radial Burst — 仕様

コンセプト1点から放射状に線が伸びる「爆発」。エネルギーの放出 = omoshikuの「面白さが広がる」瞬間。セクションごとに起点を変える。
SVG構造共通起点(cx,cy)から6-8本のlineが異なる角度・長さで放射。起点にcircle(r:5)ノード。
ストローク1px。amber/rose/blue の3色交互。opacity 0.2-0.3（繊細さ重視）。
タイミング0.4-0.8s（線の長さに比例）。0.06-0.08s刻みのstagger。ノードは即座に出現。
面白さ / テクノロジー感★★★★★ / ★★★★ — 最もエネルギッシュ。花火/爆発。omoshikuの「熱狂」を最も直接的に表現。

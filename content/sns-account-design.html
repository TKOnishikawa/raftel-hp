5アカウント×不安解消リレー

Introduction

SNS設計で考えるべき「7つの論点」

起業したらSNSを始めるのは当然。でも「何をどこで発信するか」を曖昧なまま始めると、労力だけが消えていく。

この記事では、SNSアカウント設計で考えるべき全体像と、各論点の具体的な結論を公開します。実際にRAFTELの立ち上げにあたって設計したリアルなプロセスです。

前提: この記事は「1記事から5チャネルへ — AI×SNS集客の全体設計」の続編です。コンテンツ・パイプラインの仕組みは前回で解説済み。今回はその前段となる「アカウント設計」にフォーカスします。

考えるべき論点は以下の7つ。

1

ターゲットの絞り込み

SNSで「誰に」届けるのかを1本に絞る判断

2

不安マッピング

ターゲットが抱える6つの不安を特定し、段階分けする

3

チャネル×役割の割り当て

各SNSが「どの不安を・どの段階まで」解消するかを決める

4

アカウント構成の設計

X 2アカウント体制（キャラ+実名）の狙いと使い分け

5

プロフィール設計

5アカウントそれぞれのバイオ・固定投稿・ビジュアル方針

6

導線設計（クロスリンク）

アカウント間の一方通行リンクと、フルファネルの動線

7

開設優先順位

限られたリソースで、何から始めるか

ここから各論点を順番に掘り下げます。

01 — Target

SNSのターゲットは「創業者」一本に絞る

RAFTELは創業者だけでなく、年商10〜30億円規模の中堅企業もクライアント対象です。しかしSNSで狙うのは「創業者（起業準備〜創業1年目）」に一本化しました。

SNSで狙う

創業者

起業準備中〜創業1年目の個人事業主・法人代表

  • SNSコンテンツにしやすい（自分の起業プロセスがそのままネタ）
  • 共感ベースのファン形成が自然にできる
  • 情報収集手段がSNS中心
  • 少額単価 × 多数のリード

別チャネルで狙う

中堅企業（年商10〜30億）

規模拡大フェーズでデータ活用・DXを検討中

  • 意思決定者がSNSでベンダーを探す確率が低い
  • 紹介・セミナー・SEO経由が主流
  • 高単価 × 少数の案件
  • HP記事のSEO（レイヤー2）で接点を作る

判断基準: 「そのターゲットに向けたコンテンツを毎日作れるか？」。創業者向けなら自分の起業プロセスがそのままコンテンツになる。中堅企業向けのデータ活用記事は毎日は出せない。だからSNSは創業者一本、中堅企業は別導線。

02 — Anxiety Map

創業者が抱える「6つの不安」

ターゲットが「何に悩んでいるか」を特定しないと、コンテンツは的外れになる。創業者が抱える不安を6つに分解し、それぞれの解消段階を定義しました。

心の声: 「起業したいけど、HP？ SNS？ 名刺？ 何が先？」

解消のゴール: 「なるほど、この順番でやればいいのか」と全体像が見える状態

担当チャネル: X（キャラ）のデイリー投稿で、起業ステップを小出しに発信

心の声: 「やりたいことはあるのに、人に説明できない」

解消のゴール: 「こうやって言語化すればいいのか」とプロセスが見える状態

担当チャネル: YouTube（手順動画）で、ビジョン策定の具体的手順を可視化

心の声: 「外注したら50万？ 自分で作る？ AIで作れるの？」

解消のゴール: 「AIを使えばこのクオリティが出せるんだ」と実例で確信

担当チャネル: Instagram（ビフォーアフター）＋ HP記事（詳細ガイド）

心の声: 「制作会社は高い。でもダサいHPは嫌だ」

解消のゴール: 「この価格帯でプロ品質が手に入るなら、ありかも」

担当チャネル: HP記事（費用比較・ROI証明）＋ X（実名）の権威ある解説

心の声: 「実績は？ 人柄は？ 本当にわかってるの？」

解消のゴール: 「この人は信頼できる。一緒にやりたい」

担当チャネル: X（実名）の専門見解 ＋ YouTube（人柄が伝わる動画）

心の声: 「作って終わりじゃないよね？ 3ヶ月後、自分で回せる？」

解消のゴール: 「伴走型で、自走できるまでサポートしてくれるんだ」

担当チャネル: HP記事（サービス詳細・自走マニュアルの説明）＋ YouTube（プロセス公開）

04 — Account Architecture

X「2アカウント体制」の設計思想

Xだけ2アカウント運用にした理由は、「親近感」と「権威性」を1アカウントで両立するのが不可能だからです。

キャラアカウント

ラフ（@rafu_note）

毎日投稿。起業あるある、今日の気づき、小さなTipsをカジュアルに発信。「フォローしやすい」が最優先。

  • 投稿頻度: 毎日1〜3回
  • トーン: カジュアル、共感、等身大
  • フォローの心理障壁: 低い
  • NGルール: 営業色ゼロ、リンク貼り禁止

実名アカウント

西川 貴雄（@nishikawa_raftel）

厳選投稿。業界分析・専門見解・データに基づく考察。「この人の見解を追いたい」が目標。

  • 投稿頻度: 週2〜3回
  • トーン: 知的、分析的、プロフェッショナル
  • フォローの心理障壁: やや高い（だから先にキャラで接点）
  • NGルール: 日常ツイート禁止、質で勝負

ポイントはキャラ→実名の一方通行リンク。キャラのプロフィールに「中の人 → @nishikawa_raftel」と書くが、実名側からキャラへの言及は原則しない。これにより、実名アカウントの権威性を維持します。

キャラで間口を広げ、実名で信頼を積む。
入り口は低く、出口は高く。SNS版の「フリーミアム」設計。

06 — Cross Linking

導線設計 — アカウント間のリンクルール

5アカウントをバラバラに運用しても意味がない。ユーザーが自然にファネルを進む「一方通行の導線」を設計します。

原則: 上流→下流の一方通行。カジュアルなアカウントから権威あるアカウントへ誘導する。逆方向のリンクは原則禁止（権威の希薄化を防ぐ）。

起点 誘導先 方法
X キャラ X 実名 プロフィールに「中の人 → @nishikawa_raftel」
X キャラ YouTube 「これ動画で詳しく解説したよ →」リンク付き投稿（週1回）
X 実名 HP記事 「詳細はブログで →」リンク付き投稿
YouTube HP記事 概要欄に該当記事へのリンク
YouTube X 実名 概要欄にXアカウントリンク
Instagram HP / 全リンク プロフィールのlit.link（IGはリンク1つしか貼れない）
HP記事 サービスページ 記事末尾のCTA「詳しくはこちら」
サービスページ 無料相談 ページ末尾のCTAボタン

X 実名 → X キャラへのリンクはしない。YouTube → Instagramも原則しない。リンクの本数を絞ることで、ユーザーが迷わず次のステップに進めるようにします。

07 — Priority

開設の優先順位 — 全部同時はやらない

5アカウントを同時に始めるのは非現実的。「コンテンツの源泉」に近いものから順に開設します。

1st

X キャラ（@rafu_note）

最優先 — 今すぐ開始

テキスト投稿だけで始められる。毎日の気づきを投稿するだけ。コストゼロ、学習コストゼロ。ここでコンテンツの筋トレをする。

2nd

X 実名（@nishikawa_raftel）

2週間後 — キャラの投稿リズムが安定してから

キャラ投稿の中から「深掘りすべきテーマ」が見えてくる。それを実名アカウントで専門的に展開。

3rd

YouTube（@raftel_ai）

1ヶ月後 — HP記事が3本以上溜まってから

HP記事の内容を動画化するので、まず記事のストックが必要。動画編集のワークフローも並行で構築。

4th

Instagram（@raftel.design）

1.5ヶ月後 — HP制作実績が溜まってから

ビフォーアフターの素材が必要。実績がないうちに始めてもフィードがスカスカ。「作品集」として機能し始めてから本格稼働。

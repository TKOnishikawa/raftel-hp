Phase 1 — Most Important

戦略設計 — HPの品質の8割はここで決まる

HPの見た目は後から何度でも変えられます。でも、「誰に」「何を」「なぜ伝えるか」は最初に決めないと、すべてがブレます。

このフェーズでは、AIに渡す前に「載せるべき内容」を徹底的に整理します。 地味な作業ですが、ここを飛ばして「AIでHP作って」と丸投げすると、見た目はきれいだけど中身スカスカなHPになります。

STEP 1-1 ターゲット顧客の明確化

「誰に見せるHP？」が決まらないと、何を書くかも決まりません。 「みんな」に向けたHPは、「誰にも」刺さりません。

✏ Try It — ペルソナ3行シート

以下の3行を、できるだけ具体的に埋めてください。

  1. 名前・属性: 〇〇業の経営者、従業員△名、年商□千万円（架空でOK）
  2. 今の課題: 「〜〜で困っている」「〜〜がうまくいかない」
  3. 理想の状態: あなたのサービスを受けた後、どうなっていたい？

例: 「製造業の社長、従業員20名。Web集客したいけど、HPが10年前のまま。営業に来る外注は高すぎる。自分たちの技術力が正しく伝わるHPがほしい。」

STEP 1-2 自社の強みを棚卸しする

お客さんに選ばれている理由、言えますか？
「品質が良い」「対応が早い」は強みではありません。それは当たり前です。 お客さんが「他社じゃなく御社に頼む理由」を具体的に言語化します。

✏ Try It — 強み発掘ワーク

以下の質問に答えてください（1問30秒でOK）:

  1. お客さんに「なんでうちに頼んでくれたんですか？」と聞いたら、何と言われそう？
  2. 同業他社と比べて「ここだけは負けない」と思うことは？
  3. お客さんに一番感謝されたエピソードは？
  4. 業界の「当たり前」で、自社だけ違うやり方をしていることは？
  5. もしあなたの会社がなくなったら、お客さんは何に困る？

STEP 1-3 競合サイトを分析する

同業他社のHPを3つ開いて、「うちとの違い」を明確にします。 HPの役割は「選ばれること」。選ばれるには、違いが見えなければなりません。

✏ Try It — 競合比較シート

同業他社のHPを3社ピックアップし、以下を比べてください:

  1. キャッチコピー: 何を打ち出している？
  2. サービスの見せ方: 価格表示？事例重視？プロセス説明？
  3. デザインの印象: 堅い？柔らかい？先進的？伝統的？
  4. 弱いと感じるポイント: 古い？情報不足？差別化が見えない？
  5. 自社が勝てるポイント: ここなら差をつけられる、という領域

STEP 1-4 HP全体の構成を設計する

ここまでの整理を元に、HPに載せるセクションと順番を決めます。 中小企業HPの場合、以下の構成がほぼ鉄板です。

セクション役割書くべきこと
1. ヒーロー3秒で「何の会社か」伝えるキャッチコピー + サブコピー + CTA
2. 課題共感「わかってくれる」と思わせるターゲットの悩みを3つ
3. 解決策/サービス「どう解決するか」を見せるサービス内容を3つ程度に整理
4. 強み/差別化「なぜ御社なのか」に答えるStep 1-2で整理した内容
5. 実績/事例信頼を裏付ける数字・お客様の声・Before/After
6. 代表紹介「人」を見せる経歴・想い・写真
7. お問い合わせ行動を促すフォーム or 連絡先
業種重点セクショントンマナの形容詞
コンサル / 士業代表紹介 + 実績信頼、知的、堅実
製造業技術力 + 設備 + 品質管理精密、技術、信頼
飲食 / サービス業写真 + 雰囲気 + アクセス温かみ、おいしそう、親しみ
IT / クリエイティブ事例 + 技術スタック先進的、ミニマル、スマート
医療 / 福祉安心感 + スタッフ紹介清潔、安心、やさしさ

理念/想いなぜこの事業をやるのか、どんな世界を作りたいか
事業内容具体的に何をするのか（3つ程度に整理）
差別化ポイント他社との違い、独自の手法やアプローチ
プロフィール経歴・強み・想い（実績数値があれば入れる）

ポイント: 文章の美しさは不要。箇条書きでOK。AIが整えてくれます。「伝えたい中身」だけに集中してください。

よくある落とし穴

自分の会社のことは、自分が一番客観的に見えにくい。
「うちの強みは品質です」——これ、同業他社も全員言ってます。お客さんに選ばれている本当の理由は、もっと具体的で、意外なところにあることが多い。

「当たり前だと思っていたことが、実は最大の差別化ポイントだった」という発見は、第三者との対話で頻繁に起きます。Step 1-2のワークで手が止まった方は、信頼できる人に壁打ち相手になってもらうのも手です。

Phase 1を一人でやり切る自信がある方は、Phase 2へ。

「強みの言語化が難しい」「第三者の目がほしい」と感じた方は、

Phase 2

AIツール選び & 環境構築

Phase 1で内容が整理できたら、次はAIツールを選びます。 ツール選びに迷って3日溶かす人が多いので、結論から言います。

品質重視ならこれ

Claude Code（CLI）

AIがファイルを直接作成・編集。「ここだけ変えて」が1行単位で正確に反映される。何十回修正しても他の部分が壊れない。

  • PC + Node.js + ブラウザが必要
  • AIがファイルを直接操作 → コピペ不要
  • 1,400行の中から「250行目だけ修正」が可能
  • こだわり派には圧倒的に有利

手軽さ重視ならこれ

Bolt.new / Lovable

ブラウザだけで完結。プロンプトを入れるとリアルタイムでHPが生成される。初心者に最もおすすめ。

  • インストール不要、ブラウザのみ
  • リアルタイムプレビューで直感的
  • デプロイまでワンクリック
  • 無料枠あり（月5〜10回生成程度）

Geminiでも作れますが、手動コピペが毎回必要で効率が落ちます。

初稿生成時に追加すべき指示:

HTMLファイル1つで完結する形で作成してください。 外部CSSやJSファイルは使わず、すべてインラインで記述してください。 コードは省略せず、完全な形で出力してください。

修正指示時のコツ: 「全体のコードは不要です。変更箇所のみを教えてください」と明示する。そうしないとコード全体が再生成され、修正済み箇所が意図せず元に戻ることがあります。

# 1. Claude Codeをインストール npm install -g @anthropic-ai/claude-code # 2. 作業フォルダを作成 mkdir -p output/hp # 3. 作業フォルダでClaude Codeを起動 cd output/hp claude

起動後は、チャット形式でAIに指示を出すだけ。ファイルの作成・編集・ブラウザ表示まで全て自動で行われます。

あなたの状況おすすめ
とにかく手軽に1ページ作りたいBolt.new で十分
何度も微調整して品質を追求したいClaude Code が圧倒的に効率的
コードに触ったことがないBolt.new で始めて慣れたらClaude Codeへ
複数ページや継続的な更新があるClaude Code 一択
よくある落とし穴

「どのツールがいいか」を調べ始めると、比較記事を3日読み漁って結局何も作らない——というパターンが非常に多いです。迷ったらBolt.newで始めてください。無料で、ブラウザだけで、5分で最初のHPが生成されます。

Phase 3

初稿生成 — プロンプト1発でHPを作る

Phase 1で整理した内容をAIに渡し、一気にHPを生成させます。 ここで大事なのはプロンプトの構造。「作って」と丸投げではなく、3つの要素を組み合わせます。

プロンプトの3要素: 制作指示（何を作るか）+ トンマナ（どんな雰囲気か）+ コンテンツ（Phase 1で整理した全テキスト）

以下の中小企業HPを作成して。 1ページ完結型、HTMLファイル1つで。レスポンシブ対応必須。 ■ デザインの方向性 [Phase 1-3の競合分析を踏まえたトンマナの形容詞を入れる] 例: 洗練されたデザイン、信頼感、技術力を感じるトーン ■ HP構成 [Phase 1-4で決めたセクション構成を入れる] 1. ヒーロー キャッチコピー: ○○○○ サブコピー: ○○○○ 2. 課題共感 - ○○で困っていませんか？ - ○○が思うようにいかない… 3. サービス内容 （Phase 1で整理した内容を貼り付け） ... ■ 参考サイト（あれば） https://参考サイトのURL

ポイント:

・ Phase 1の成果物をそのままプロンプトに貼り付けるのが最も効果的

・トンマナは形容詞で伝える（「洗練」「温かみ」「信頼感」等）

・参考URLを添えるとAIがデザインの方向性を理解しやすい

この段階で得られるのは「70点のたたき台」です。 完璧を求めず、まずは形にすることが重要。ここからPhase 4で磨いていきます。

よくある落とし穴

プロンプトに入れる情報の質 = Phase 1の質です。
Phase 1を適当にやると、ここでAIが生成するHPも「きれいだけど中身が薄い」ものになります。「AIが微妙なHP作った」と感じたら、原因の9割はPhase 1にあります。

Phase 4

ブラッシュアップ — 70点を100点にする

初稿をブラウザで見て、段階的に調整していきます。 「大→小」の順で修正するのがコツ。最初に細部をいじると、全体を変えたときに巻き戻されます。

STEP 4-1 大方針の調整（全体の方向性）

色・フォント・レイアウト・雰囲気など「全体の方向性」を調整します。1回の指示で3〜5個に絞るのがコツ。

1. 良い点を先に伝える — 「すばらしい」と言うことで、AIが良い部分を壊さなくなる。

2. 変えたい方向を"感覚"で伝える — 「脳に入ってくる」「うるさくない」など。技術用語は不要。

3. トレードオフを明示する — 「明るくしたい。でも洗練は維持して」のように、守りたいラインを伝える。

4. 一度に3〜5個に絞る — 多すぎると品質が落ちる。大きな修正は回数を分ける。

すばらしい。version2として以下のチューニングversionを作成して。 ・4つのアプローチのところをインフォグラフィックで、 　もっと感覚的に脳にはいってくる感じにして。 ・黒すぎるので、もう少し明るいトーン。 　だけど洗練・エネルギー感・有機的。 ・エネルギーがみなぎっているような背景アニメいいね。 　もう少しだけ目立たせて。ただ、うるさくはしたくない。

STEP 4-2 ディテール調整（セクション単位）

全体の方向性が決まったら、セクションごとに1つずつ磨き込みます。 「〇〇セクションの△△を□□にして」と、場所→対象→変更内容の3点を明確に。

指示何が変わったか
「ヒーローの背景をもう少し明るく」ダーク黒 → ウォームベージュに
「サービスセクションをカード型にして」リスト → 3カラムカードに
「CTAボタンを中央寄せにして」text-align: center適用
「代表紹介に写真スペースを追加」2カラムレイアウト追加
よくある落とし穴

AIが出した70点を100点にするには、マーケティングの視点が必要です。
「何が70点で何が100点か」は、正直マーケ経験がないと判断しづらい部分があります。特に「キャッチコピーの刺さり具合」「CTAの配置と文言」「ファーストビューの情報優先度」は、経験がものを言う領域です。
ここは自分で何案か試してみて、知人やお客さんに見てもらうのが効果的です。

Phase 5

レスポンシブ & 公開

最後にスマホ表示を確認し、微調整して公開します。 HPを見る人の6〜7割はスマホ。PC表示だけで満足しないでください。

問題AIへの指示の出し方
テキストが変な位置で改行される「スマホの場合、○○の改行位置を調整して。iPhone（横幅393px）基準で」
ボタンが左に寄っている「スマホでは○○ボタンを中央寄せにして」
要素同士が重なっている「スマホで○○と△△が被っている。余白を調整して」
文字が小さすぎる「スマホでは本文を16px以上にして」

確認方法: ブラウザのDevTools（F12）→ スマホ幅をシミュレート、またはスマホ実機で確認。

AI×プロンプトだけで
「プロ品質のHP」を作る完全ガイド

コーディング知識ゼロでも、AIとの対話だけで洗練されたホームページを作れる時代。
実際にRAFTEL HPを制作したプロセスとプロンプトを全公開します。

2026.02 Reading time: 8 min

Overview

全体像: 5つのフェーズ

HP制作は「一発で完成」ではなく、AIとの対話を重ねて磨き上げる反復プロセスです。 料理でいえば、レシピ（素材）を用意し、AIに調理してもらい、味見しながら調整していく流れです。

1

事前準備

あなたがやること: コンテンツ素材 + 参考サイト + 環境構築

HPに載せるテキスト・参考デザイン・AIツールを準備する。ここが品質の9割を決める。

2

初稿生成

AIがやること: HTML/CSS/JS一式を自動生成

プロンプト1発で完全なHPが生成される。まずは70点のたたき台を手に入れる。

3

大方針のブラッシュアップ

あなた→AI: トンマナ・構造レベルの修正指示

色・フォント・レイアウトなど「全体の方向性」を調整する。細部はまだ触らない。

4

ディテール調整

あなた→AI: セクション単位の磨き込み

各セクションを1つずつ見ながら「ここをこう変えて」とピンポイント修正。

5

レスポンシブ & 仕上げ

あなた: スマホ実機確認 → AI: レスポンシブ対応

スマホ・タブレットでの表示崩れを修正し、最終仕上げ。

Phase 1

事前準備 — AIに渡す「素材」を揃える

AIに丸投げしても「それっぽいけど中身スカスカ」なHPになる。AIに渡す前の素材の質が、アウトプットの9割を決める。 やるべきことは3つ。

1. テキストコンテンツを書き出す

HPに載せたい情報を、箇条書きでいいので全部書き出す。文章の美しさはAIが整えてくれるので、伝えたい「中身」だけに集中する。

セクション書くべきこと
キャッチコピー一言で「何者か」が伝わるフレーズ
サブコピーキャッチの補足（誰に・何を・どう届けるか）
理念なぜこの事業をやるのか、どんな世界を作りたいか
事業内容具体的に何をするのか（3つ程度に整理）
差別化ポイント他社との違い、独自の手法やアプローチ
プロフィール経歴・強み・想い（人材紹介系なら実績数値も）

今回のRAFTELの場合: Vision/Mission/Value の理念体系、3つの事業内容、4つのアプローチ、代表略歴・強み・メッセージを約2,000字で書き出した。

2. 参考サイトを見つける

「こんな雰囲気にしたい」というサイトを1〜3個見つけておく。AIにURLを渡すと、そのトンマナを参考にしてくれる。

全体のトーン: ダーク/ライト、シャープ/温かみ、ミニマル/リッチ

レイアウト構成: 1カラム/2カラム、カード型/リスト型、余白の取り方

アニメーションの有無・強度: 静的/スクロール連動/パーティクル等

フォントの雰囲気: ゴシック系/明朝系/手書き風、太さのメリハリ

今回の参考: ecfs.jp — 洗練されたダークベース＋温かみのあるアクセントカラーが、RAFTELの「データ×魂」というコンセプトにマッチしていた。

3. AIツール & 環境を用意する

2パターンから選べる。何度も調整を重ねたい場合はClaude Codeが圧倒的に効率的。

Recommended

パターンA: Claude Code（CLI）

AIがファイルを直接作成・編集。「ここだけ変えて」が1行単位で正確に反映される。

  • PC + Node.js + ブラウザが必要
  • AIがファイルを直接操作 → コピペ不要
  • 1,400行の中から「250行目だけ修正」が可能
  • 何十回修正しても他の部分が壊れない

Easy Start

パターンB: Gemini（ブラウザ版）

インストール不要、ブラウザだけで始められる。ただし手動コピペが毎回必要。

  • Googleアカウント + ブラウザのみ
  • AIがコードを出力 → 自分でコピペ保存
  • 修正のたびにコード全体が再出力されがち
  • 長いコードは途中で切れる場合あり
# 1. Claude Codeをインストール npm install -g @anthropic-ai/claude-code # 2. 作業フォルダを作成 mkdir -p output/hp # 3. 作業フォルダでClaude Codeを起動 cd output/hp claude

起動後は、チャット形式でAIに指示を出すだけ。ファイルの作成・編集・ブラウザ表示まで全て自動で行われる。

初稿生成時に追加すべき指示:

HTMLファイル1つで完結する形で作成してください。 外部CSSやJSファイルは使わず、すべてインラインで記述してください。 コードは省略せず、完全な形で出力してください。

Geminiは親切心で「省略」したり「ファイル分割」を提案することがある。単一HTMLで完結させないと管理が煩雑になる。

修正指示時のコツ: 「全体のコードは不要です。変更箇所のみを教えてください」と明示する。そうしないとコード全体が再生成され、修正済み箇所が意図せず元に戻ることがある。

工程Claude CodeGemini
初稿生成AIがファイル直接作成コピペして保存が必要
修正該当箇所だけ差分編集コード全体が再出力されがち
長いコード1,400行でも問題なし途中で切れる場合あり
修正の安全性他の箇所が壊れない意図しない変更リスクあり

Phase 2

初稿生成 — プロンプト1発でHPを作る

準備した素材をAIに渡し、一気にHPを生成させる。ここで大事なのはプロンプトの構造。 「作って」と丸投げではなく、3つの要素を組み合わせる。

プロンプトの3要素: 制作指示（何を作るか）+ トンマナ（どんな雰囲気か）+ コンテンツ（中身の全テキスト）

以下のHPを作成して。 洗練されたデザイン、熱量を感じさせるパッションで。 ホームページ構成案 1. ヒーローメッセージ（Top View） キャッチコピー: ○○○○○ サブコピー: ○○○○○ 2. 理念体系 (Philosophy) Vision: ○○○○○ Mission: ○○○○○ ... （Phase 1で準備した全テキストを貼り付け） 以下HPは参考： https://参考サイトのURL

ポイント:

・「作成して」と明確に指示する（「考えて」ではなく）

・トンマナは形容詞で伝える（「洗練」「熱量」「有機的」「信頼感」等）

・コンテンツは構造化して渡す（セクション分け + 見出し + 本文の階層）

・参考URLを添える → AIがデザインの方向性を理解する手がかりになる

業種トンマナの形容詞参考にすべきサイトの傾向
コンサル / データ系洗練、熱量、エネルギー、有機的ダーク or ベージュ系 + アクセントカラー
人材紹介信頼感、温かみ、親しみやすさ、プロフェッショナルブルー系 + 人物写真 + 明るいトーン
クリエイティブ / デザインモダン、ミニマル、アーティスティック、大胆モノトーン + 大きなビジュアル + 余白
飲食 / ウェルネスナチュラル、安心感、上質、健やかさグリーン or アーストーン + テクスチャ

この段階で得られるのは「70点のたたき台」。 RAFTEL HPの場合、初稿（V1）は約1,500行のHTML/CSS/JSが一発で生成された。 ダーク背景 + パーティクルアニメーション + 全セクション揃った状態で、ここからブラッシュアップを重ねていく。

Phase 3

大方針のブラッシュアップ — 全体の方向性を調整

初稿をブラウザで見て、「方向性」レベルの調整を指示する。ここではまだ細部に触れない。 「色が暗すぎる」「もっと有機的に」「カード型にしたい」など、大枠の修正を1回の指示にまとめる。

すばらしい。version2として以下のチューニングversionを作成して。 ・4つのアプローチのところをインフォグラフィックで、 　もっと感覚的に脳にはいってくる感じにして。 ・事業内容のところも、インフォグラフィックで、感覚的に。 ・黒すぎるので、もう少し明るいトーン。 　だけど洗練・エネルギー感・有機的。 ・エネルギーがみなぎっているような背景アニメいいね。 　もう少しだけ目立たせて。ただ、うるさくはしたくない。

このフィードバックで変わったこと:

・背景: ダーク黒 → ウォームベージュ（#f6f3ee）に大転換

・フォント: Sora（ヘッダー用）、Zen Antique（筆文字風）を追加

・アプローチ: テキストカード → 写真付きシネマティックカード

・事業内容: リスト型 → 黒背景+写真ヘッダーカード

・パーティクル: 30個→50個、色バリエーション追加

1. 良い点を先に伝える — 「すばらしい」と言うことで、AIが良い部分を壊さなくなる。

2. 変えたい方向を"感覚"で伝える — 「脳に入ってくる」「うるさくない」など。技術用語は不要。

3. トレードオフを明示する — 「明るくしたい。でも洗練は維持して」のように、守りたいラインを伝える。

4. 一度に3〜5個に絞る — 多すぎると品質が落ちる。大きな修正は回数を分ける。

Phase 4

ディテール調整 — セクション単位で磨く

全体の方向性が決まったら、セクションごとに1つずつ見ながら磨き込む。 1回の指示で1〜3箇所が理想。ブラウザで確認→指示→確認のループを回す。

指示何が変わったか
「Visionを筆文字フォントで表現して」Zen Antique適用、背景画像除去
「Missionは画像に差し替えて」テキスト → 画像ブロックに変更
「事業内容の背景を黒にして」ダーク背景+アンビエントグロー追加
「CTAボタンを中央寄せにして」wrapper div追加でtext-align: center
「スマホではタイトルだけ中央寄せに（PCは左）」768pxメディアクエリに条件追加

コツ: 「〇〇セクションの△△を□□にして」と、場所→対象→変更内容の3点を明確にする。曖昧な指示は意図しない変更を招く。

Phase 5

レスポンシブ & 仕上げ — スマホで確認

最後にスマホ表示を確認し、微調整する。 ブラウザのDevTools（F12）でスマホ幅をシミュレートするか、実機で確認する。

問題指示の出し方
テキストが変な位置で改行される「スマホの場合、○○の改行位置を調整して。iPhone 17（横幅393px）使用」
ボタンが左に寄っている「スマホでは○○ボタンを中央寄せにして」
要素同士が重なっている「スマホで○○と△△が被っている。余白を調整して」
PC/スマホで異なる配置にしたい「スマホでは中央寄せ、PCでは左寄せにして」と明示

今回の例: ヒーローキャッチの改行問題は、.br-desktop / .br-mobile というクラスで「PCでは改行、スマホでは改行しない」をCSSで制御する方法で解決した。

AIに「何を作るか」は人が決め、
「どう作るか」はAIに任せる。

業種やトンマナが違っても、このフレームワークは同じ。
素材の質 × フィードバックの精度 = アウトプットの品質。

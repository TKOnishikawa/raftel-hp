# RAFTEL HP ライティングガイド

## 目的

このガイドは、RAFTEL HPの一貫したトーン・構造・表現を維持するためのライティングルールを定義します。

---

## 1. トーン・文体

### 1.1 基本方針

**ターゲット**: 起業家・経営者（創業期〜成長企業）

**トーン**: プロフェッショナル × 親しみやすさ

- **形式的すぎない**: 「〜です・ます」調だが、硬くない
- **共感を示す**: 課題への理解を示してから解決策を提示
- **熱量を伝える**: 「想い」「熱狂」「確信」など、エネルギーのある言葉を使う

---

### 1.2 トーンの使い分け

| ページカテゴリ | トーン | 文体例 |
|--------------|--------|--------|
| **HOME (index.html)** | フォーマル | 「〜が〜」「〜である」 |
| **Service Pages** | フォーマル | 「〜が〜」「〜である」 |
| **Content Pages** | カジュアル | 「〜けど〜」「〜だが〜」（適度に） |

**重要**: 2024年以降、全ページで「〜が〜」（形式的）に統一しました。「〜けど〜」（親密的）は使用しません。

#### 例: Principle Box

**Before (親密的・不統一)**:
```
コーチに頼むと、言葉にはなるけど形にならない。
制作会社に頼むと、形にはなるけど想いが抜ける。
```

**After (形式的・統一)**:
```
コーチに頼むと、言葉にはなるが形にならない。
制作会社に頼むと、形にはなるが戦略が抜ける。
```

## 2. 構造・フォーマット

### 2.1 基本フロー: Problem → Solution → Proof → CTA

全Serviceページは以下の構造に従います:

1. **Problem (課題提示)**: ターゲットが抱える具体的な課題
2. **Solution (解決策)**: RAFTELが提供する価値・アプローチ
3. **Proof (根拠・実績)**: RTB (Reason to Believe) - 実績、比較表、Personal Why
4. **CTA (行動喚起)**: 価格 + 次のアクション

---

### 2.2 Problem Section

**書き方**:
- **具体的なシーンを描く**: 「〜したいが、〜できない」
- **共感を示す**: 「その気持ち、わかります」を暗に示す
- **3-6個の課題を提示**: 多すぎると散漫、少なすぎると刺さらない

**例**:
```html
<h4>「想い」はあるのに、<br>伝え方がわからない</h4>
<p>起業の熱量はある。でもHPの文章が書けない。SNSで何を発信すればいいかわからない。</p>
```

**NG例**:
```html
<h4>マーケティングが苦手</h4>
<p>マーケティングスキルがない人は困ります。</p>
```
→ 抽象的すぎる、共感がない

---

### 2.3 Solution Section

**書き方**:
- **「何を」「どうやって」を明確に**: 抽象論ではなく具体的な手段
- **RAFTELならではの強み**: 他と何が違うのか
- **3つのポイントで提示**: 認知負荷を減らす

**例**:
```html
<h3>1. ビジョン言語化</h3>
<p>1.5時間の壁打ちで、あなたの想いを言語化。競合と差別化できる「強み」を見つけます。</p>
```

**NG例**:
```html
<h3>ビジョン策定</h3>
<p>ビジョンを作ります。</p>
```
→ 「どうやって」が不明確

---

### 2.4 Proof Section

**RTB (Reason to Believe)** で信頼を獲得:

1. **数字**: 年4000万円2年継続、10億円規模、LTV 1.4x
2. **比較表**: コーチ・コンサル・制作会社との違い
3. **Personal Why**: なぜRAFTELを作ったのか

**書き方**:
- **具体的な数字**: 「大きな成果」ではなく「年商+7億円」
- **ストーリー**: 数字だけでなく、どう達成したか
- **共感**: 「あなたと同じ課題を感じていた」

---

### 2.5 CTA Section

**書き方**:
- **価格を明示**: 不透明感を排除
- **次のアクションを明確に**: 「お問い合わせ」「資料ダウンロード」
- **後押しの一言**: 「まずはここから」「一緒に創っていく」

**例**:
```html
<p class="cta-price">&yen;30,000</p>
<h3 class="cta-title">まずはここから。3日でプロ品質のHPを。</h3>
<a href="#contact" class="btn btn--primary">お問い合わせ</a>
```

## 3. キャッチコピー・ブランドメッセージ

### 3.1 コアメッセージ

**RAFTEL の存在理由**:
```
想いを、仕組みに。
事業の成長と熱狂を「仕組み」で一気通貫で構築します。
```

**Personal Why**:
```
AI技術の進化により、ビジネスのあり方は劇的に変わりつつあります。
しかし、最後に人を動かし、事業を動かすのは「熱狂」や「確信」といった人間ならではのエネルギーです。

戦略だけでは動かない。実装だけでは方向を見失う。想いだけでは形にならない。
コーチング、コンサルティング、エンジニアリング — この3つを横断しながら、
「現場」と「人」にフォーカスし、事業の成長と熱狂を一緒に創っていく。
それがRAFTELの存在理由です。
```

---

### 3.2 サービス別キャッチコピー

| サービス | キャッチコピー | 補足説明 |
|---------|--------------|---------|
| **Quick Launch** | まずはここから。3日でプロ品質のHPを。 | エントリープラン・スピード感 |
| **Growth Partner** | 想いを、仕組みに。3ヶ月で事業の土台を整える。 | 創業期・伴走型 |
| **Growth Advisory** | データ駆動で、事業の成長を加速する。 | 成長企業・KPI最適化 |

## 4. 数字の表現ルール

### 4.1 実績の表現

| 数字 | 表現方法 | 用途 |
|------|---------|------|
| **年4000万円2年継続** | 「年4,000万円で2年更新」 | プロフィール統計 |
| **月340万** | 「月340万円」（括弧内で補足） | 補足説明 |
| **+7億円** | 「YoY -4% → +35%のV字回復」 | 成長率と併記 |
| **LTV 1.4x** | 「LTV 1.4倍に改善」 | 改善幅を明示 |

**ルール**:
- **カンマ区切り**: 4桁以上は「4,000」
- **単位を明示**: 「円」「%」「倍」
- **文脈を添える**: 「どんな状況で」「どう達成したか」

---

### 4.2 価格の表現

| サービス | 価格表記 | 補足 |
|---------|---------|------|
| Quick Launch | &yen;30,000 | 初稿納品まで |
| Growth Partner | &yen;200,000/月 &times; 3ヶ月 | 伴走型 |
| Growth Advisory | &yen;300,000〜 | 月次顧問 |

**ルール**:
- **HTMLエンティティ**: `&yen;` を使用
- **期間を明示**: 「/月」「&times; 3ヶ月」
- **不確定要素**: 「〜」で幅を示す

---

## 5. 避けるべき表現

### 5.1 抽象的な表現

❌ **NG**: 「最高のソリューションを提供します」
✅ **OK**: 「1.5時間の壁打ちで、想いを言語化します」

❌ **NG**: 「豊富な実績」
✅ **OK**: 「年4,000万円で2年更新、10億円規模の新規事業立ち上げ」

---

### 5.2 他社批判

❌ **NG**: 「他社はできないが、RAFTELならできる」
✅ **OK**: 「コーチに頼むと言葉にはなるが形にならない。RAFTELは全部一人でやります。」

→ 事実ベースの比較はOK、感情的な批判はNG

---

### 5.3 過度な謙遜

❌ **NG**: 「微力ながらお手伝いします」
✅ **OK**: 「事業の成長と熱狂を一緒に創っていく」

→ プロフェッショナルとして自信を持つ

## 7. チェックリスト

新規ページ・コンテンツ作成時のチェックリスト:

- [ ] トーンは「〜が〜」（形式的）で統一されているか
- [ ] Problem → Solution → Proof → CTA の流れになっているか
- [ ] 具体的なシーン・数字が含まれているか
- [ ] 抽象的な表現（「最高」「豊富」等）を避けているか
- [ ] キャッチコピーがサービスの本質を表しているか
- [ ] 数字にカンマ区切り・単位が正しく表記されているか
- [ ] 他社批判ではなく、事実ベースの比較になっているか
- [ ] CTAが明確で、次のアクションが分かりやすいか
